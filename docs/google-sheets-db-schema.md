## Tables

Columns:

- `key`
- `value`
- `description`
- `updated_at`

Seeded keys:

- `company_domain`: `kyanon.digital`
- `admin_email`: `hau.nt@kyanon.digital`
- `auth_provider`: `google_workspace`
- `timezone`: `Asia/Saigon`
- `registration_deadline_rule`: `LAST_DAY_PREVIOUS_MONTH`
- `default_weekend_role`: `QC/PO`
- `default_weekend_required_count`: `1`
- `default_day_hours`: `8`

- `google_chat_reminder_days_before`: `1`

### `users`

Stores users allowed by the app.

Columns:

- `user_id`
- `email`
- `username`
- `display_name`
- `role`
- `status`
- `source`
- `created_at`
- `updated_at`

Seeded from old spreadsheet headers. `hau.nt@kyanon.digital` is `ADMIN`; all other seeded users are `MEMBER`.

Admin user management rules:

- Add user: append a new row to `users`.
- Required fields for new user:
  - `email`
  - `display_name`
  - `role`
  - `status`
- `email` must use the `@kyanon.digital` domain.
- `email` must be unique case-insensitively.
- `username` is derived from the email local part unless admin overrides it.
- `user_id` is generated by app code from the normalized username.
- New users default to `MEMBER` and `ACTIVE`.
- Remove user: set `status` to `INACTIVE` and update `updated_at`.
- Hard delete is allowed only when the user has no rows in `registrations`, `update_requests`, or `audit_logs`.
- The admin account `hau.nt@kyanon.digital` cannot be removed or deactivated by the app.
- Every add/remove/deactivate action must append a row to `audit_logs`.

## App Write Rules

- Treat row 1 as immutable headers.
- Do not let normal users edit the DB file directly.
- App server reads/writes the sheet through Google Sheets API using a service account.
- The Google Sheet must be shared with the service account `client_email` as Editor.
- Generate stable IDs in app code, for example `slot_2026_06_06`, `cap_2026_06_06_qc_po`, `reg_<timestamp>_<username>`.
- Before creating a registration, app code must:
  - Check the user is active.
  - Check the slot is `OPEN`.
  - Check remaining capacity.
  - Check the weekend one-day-per-month rule.
  - Re-read target rows immediately before append.
- Store all timestamps in ISO format.
- User management writes must:
  - Re-read `users` before writing.
  - Enforce unique email.
  - Refuse changes that would remove/deactivate the configured admin email.
  - Append audit logs for add, deactivate, reactivate, and hard delete actions.
