# OT Support Tracking Tool Requirements

## Product Goal

Build a web app to manage weekend and holiday OT support registration for a project team, with Google Workspace login, admin-controlled schedule setup, monthly dashboard, Excel export, Google Chat reminder, and correction requests.

## Confirmed Decisions

- Login: Google Workspace.
- Access control: only `kyanon.digital` accounts can log in.
- Admin: one admin account only, `hau.nt@kyanon.digital`.
- User registers by clicking a slot in the Dashboard calendar and confirming the popup.
- User can cancel their registered future weekend slot by clicking their registered calendar slot and confirming the popup.
- User can change their registered future weekend slot by cancelling first, then selecting another available slot.
- Member cannot register, cancel, or edit past slots directly.
- Admin can register past slots on behalf of members for correction/backfill.
- Past dates cannot be changed directly by users. Users must submit an update request for admin review.
- Registration rule: each person can have one active support registration per month.
- One support day equals 8 hours.
- No team/project segmentation is needed for MVP.
- Registration deadline: the last day of the previous month. If a member does not register before the deadline, admin can assign a member manually.
- Weekend schedules are auto-generated by month.
- Holiday/Tet schedules are configured by admin by single date or date duration.
- If an admin holiday/Tet setting overlaps an auto-generated weekend date, the admin setting takes priority.
- Default weekend capacity: Saturday and Sunday each require one person for `QC/PO`.
- Dashboard needs both calendar view and table view.
- Dashboard display name: `Register Dashboard`.
- Slot information shown on dashboard:
  - Date
  - Day of week
  - Type: weekend, holiday, Tet
  - Registered accounts
  - Remaining slots
  - Approval status
- Excel export is monthly.
- Excel export must preserve the provided template format, including logo, formulas, and approval/signature layout.
- Excel template: `templates/AMAZE _ Time log - Overtime - 2026.xlsx`.
- Template workbook sheets:
  - `Jan2025` hidden
  - `Feb-Apr2025`
  - `May-Nov2025`
  - `Dec2025`
  - `Jan-Mar2026`
  - `Apr-May2026`
  - `Feb2025` hidden
- Google Chat reminder: send before the support day by 1 day, in Vietnamese with accents.
- Google Chat message example: `Ngày 2026-06-15: Nguyễn Văn A sẽ support SOS dự án X`
- User update request fields:
  - Date to update
  - Reason
  - Actual man/month or hours
  - Evidence file or note
- Admin can approve/reject requests.
- Request history must be stored.
- Deployment target: Render.
- Primary storage for MVP: Google Sheets, spreadsheet `OT Support Tool DB`.
- Google Sheets DB URL: `https://docs.google.com/spreadsheets/d/1ts9sj3cdT_Z3BMrtHo2_R5H35gxPmRlnztXxLkM8fog/edit`.
- Google Sheets DB ID: `1ts9sj3cdT_Z3BMrtHo2_R5H35gxPmRlnztXxLkM8fog`.
- Local app reads/writes Google Sheets DB through the Node server API. Browser code must not receive Google credentials.
- Registrations are auto-approved.
- Holiday/Tet slots can have multiple registered users on the same date, up to configured capacity.
- Holiday/Tet man/month factor is divided equally among active registered users for that slot.
- Members cannot see `Schedule` and `Users` menus.

## Admin Capabilities

- Configure holiday/Tet support days by date duration.
- Set start date, end date, holiday name, required headcount, man/month factor, and note.
- Create schedule slots without assigning members immediately.
- Member registration is handled from the Dashboard calendar popup. The old Register menu is removed.
- Register past slots on behalf of members.
- Add member accounts.
- Remove member accounts by marking them inactive in the Google Sheets DB.
- Hard-delete a member row only when that member has no registrations, update requests, or audit history.
- Review and approve/reject update requests.
- Export monthly schedule to Excel.
- Review statistics by person.

## User Capabilities

- Log in with Google Workspace account.
- Register for available weekend slots.
- View dashboard calendar and table.
- View personal support statistics.
- Compare tracked OT with actual work.
- Submit correction/update requests.

## Capacity Model

Each schedule date can have multiple role-based capacity entries.

Examples:

- PO: configured by admin.
- QC: configured by admin.
- DEV: configured by admin.

For MVP, capacity should support:

- Role name.
- Required number of people.
- Registered people.
- Remaining slots.
- Per-slot hours or man/month factor.

## Storage Note

For MVP, the app will use a native Google Sheets spreadsheet as the primary storage layer instead of PostgreSQL.

This is acceptable for a small internal tool, but app code must enforce integrity rules because Google Sheets is not a relational database.

Important safeguards:

- Only the app/admin should edit the DB spreadsheet structure.
- Member-facing changes should go through the app.
- Admin user management changes must be written back to the `users` sheet.
- Registration uniqueness and capacity checks must be implemented in app code.
- Writes must re-read the target rows before committing to reduce conflicting updates.
- Local and deployed server need Google API credentials with edit access to the DB spreadsheet.
- If usage grows, migrate to PostgreSQL later.

## Proposed MVP Stack

- Framework: Next.js.
- Auth: Google OAuth / Google Workspace domain restriction.
- Storage: Google Sheets API using spreadsheet `OT Support Tool DB`.
- UI: Calendar monthly view, slot table, admin setup pages, request management page, personal statistics page.
- Excel: Template-based `.xlsx` export.
- Notification: Google Chat webhook job, initially configurable and disabled until webhook is available.
